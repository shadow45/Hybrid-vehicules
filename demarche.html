Methodologie de travail

La méthode choisie au cours du développement de notre projet SID est « la méthode agile XP » La méthode XP (eXtreme Programming) définit un certain nombre de bonnes pratiques permettant de développer un logiciel dans des conditions optimales en plaçant le client au cœur du processus de développement, en relation étroite avec le client. L'eXtreme Programming est notamment basé sur les concepts suivants :

  • Les équipes de développement travaille directement avec le client sur des cycles très courts d'une à deux semaines maximum.
  • Les livraisons de versions du logiciel interviennent très tôt et à une fréquence élevée pour maximiser l'impact des retours utilisateurs.
  • L'équipe de développement travaille en collaboration totale sur la base de binômes.
  • Le code est testé et nettoyé tout au long du processus de développement.
  • Des indicateurs permettent de mesure l'avancement du projet à fin de permettre de mettre à jour le plan de développement.

Collecte d'informations

Afin de collecter les données , il faut cerner le domaine de recherche en se limitant sur 3 veilles à savoir:

  • Veille technologique
  • Veille environnement
  • Veille presse

Veille technologique

Dans cette veille la principale source d'informations était le site espacnet qui donne la possibilité d'étudier les brevets sur les véhicules hybrides.

Veille envrinnoment

Dans cette veille , la principale source d'informations sont les différentes pages html sur le web .

Veille presse

Concernant la veille presse , on s'est rendus sur les différents sites de la presse marocaine et internationnalle afin de collecter le maximum d'informtions concernant les véhicules hybrides.

Traitement des informations

Dans la phase de traitement , on s'intéresse à éliminer toutes les redondances, extraire les mots clés, rafiner les données obtenues afin d'avoir des données fiables choses qui est faite en utilisant pas mal d'outils .

Alimentation de la base de données

Une fois l'étape de traitement de données est achevé , on a préparé un code java qui permet d'alimenter la base de données en remplissant tous les champs avec les informations adéquates en respectant les dépendances entre les champs et les tables.
